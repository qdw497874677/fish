# 《水族守卫》MVP 游戏设计文档

## 1. 项目定位

《水族守卫》是一款 2D 点击式水族箱经营 + 即时防守游戏。玩家通过投喂鱼群、收集金币、购买鱼和升级项目来扩大鱼缸经济，同时抵御入侵生物，最终购买关卡目标物完成通关。

本项目借鉴的是“养成经济循环 + 入侵防守打断”的玩法类型，不复制《疯狂水族馆》的具体角色、美术、名称、音效、UI 表达和关卡数值。

## 2. 平台目标

- 首发平台：网页版 Godot Web 导出
- 输入方式：鼠标点击与移动端触摸兼容
- 后续方向：如果继续迁移小游戏，需要另行评估平台适配成本

## 3. MVP 目标

第一版目标是做出一个 5-10 分钟可玩的闭环 Demo，而不是完整商业版本。

玩家应能完成以下流程：

```text
进入关卡 → 投喂鱼 → 鱼成长 → 鱼产金币 → 点击收金币
→ 购买更多鱼/升级食物 → 敌人入侵 → 点击攻击敌人
→ 攒钱购买 3 个水晶核心 → 通关 → 解锁助手
```

## 4. 核心玩法循环

### 4.1 经营循环

```text
投喂 → 鱼吃食物 → 饱食/成长 → 定时产金币 → 玩家收集 → 购买/升级
```

### 4.2 防守循环

```text
入侵预警 → 敌人出现 → 敌人追逐鱼 → 玩家点击攻击 → 击退敌人 → 安全奖励窗口 → 恢复经营
```

### 4.3 通关循环

```text
积累金币 → 购买水晶核心 1 → 购买水晶核心 2 → 购买水晶核心 3 → 通关结算
```

## 5. 核心实体

### 5.1 鱼 Fish

鱼是主要经济来源。

属性：

- 生命状态：存活 / 死亡
- 成长阶段：幼年 / 成熟
- 饥饿值：随时间降低
- 游动速度
- 当前目标：游动目标 / 食物目标 / 逃离敌人
- 产出间隔
- 产出金币类型

MVP 鱼类：

| 名称 | 价格 | 成长时间 | 成熟产出 | 饥饿死亡时间 | 说明 |
|---|---:|---:|---:|---:|---|
| 蓝泡鱼 | 100 | 20 秒 | 15 金币 / 8 秒 | 25 秒 | 基础鱼，吃普通鱼食 |
| 红鳍鱼 | 500 | 30 秒 | 80 金币 / 12 秒 | 30 秒 | 高级鱼，吃小型蓝泡鱼或高级食物 |

第一版可以先只实现蓝泡鱼，红鳍鱼作为第二关内容加入。

### 5.2 食物 Food

玩家点击水体区域投放食物。食物从点击位置开始下沉，鱼会寻找最近的可食用食物。

属性：

- 下沉速度
- 营养值
- 价格
- 是否被鱼锁定
- 触底后消失时间

食物等级：

| 等级 | 单价 | 效果 |
|---|---:|---|
| Lv1 | 5 | 基础饱食与成长 |
| Lv2 | 10 | 成长效率 +50% |
| Lv3 | 20 | 成长效率 +100% |

### 5.3 金币 Coin

鱼成熟后定时掉落金币。金币缓慢下沉，玩家点击后收入账户，沉到底部一段时间后消失。

属性：

- 金额
- 下沉速度
- 生命周期
- 是否已被收集

### 5.4 敌人 Enemy

敌人定时入侵。入侵时玩家的主要任务从经营切换为防守。

MVP 敌人：

| 名称 | 血量 | 行为 | 设计目的 |
|---|---:|---|---|
| 小章怪 | 5 | 追逐最近的鱼，接触后吃掉 | 基础威胁，教会玩家点击攻击 |
| 甲壳怪 | 10 | 移动慢，血量高 | 压力测试，要求玩家集中火力 |

### 5.5 助手 Pet

助手是长期解锁内容，不需要玩家直接控制。

MVP 助手：

| 名称 | 能力 | 解锁方式 |
|---|---|---|
| 清洁螺 | 自动捡起底部附近金币 | 第 1 关通关 |
| 泡泡海马 | 每隔一段时间自动给饥饿鱼投喂一次，食物较多时不会继续堆叠 | 第 2 关通关 |
| 电光水母 | 敌人出现时周期性自动电击最近敌人，造成少量伤害 | 第 3 关通关 |

## 6. 关卡设计

MVP 包含 3 个关卡。

| 关卡 | 新内容 | 通关目标 | 敌人 | 助手奖励 |
|---|---|---|---|---|
| 1 | 蓝泡鱼、投喂、金币、普通敌人 | 买 3 个水晶核心 | 小章怪 | 清洁螺 |
| 2 | 食物升级、红鳍鱼 | 买 3 个水晶核心 | 小章怪 | 泡泡海马 |
| 3 | 更高价格目标、血厚敌人 | 买 3 个水晶核心 | 小章怪 + 甲壳怪 | 电光水母 |

## 7. UI 结构

游戏主界面分为三层：

```text
┌────────────────────────────────────┐
│ 顶部商店栏：鱼 / 食物升级 / 水晶核心 │
├────────────────────────────────────┤
│                                    │
│            水族箱主玩法区域          │
│     鱼、食物、金币、敌人、助手       │
│                                    │
├────────────────────────────────────┤
│ 金币数量 / 关卡目标 / 暂停按钮       │
└────────────────────────────────────┘
```

MVP 必备 UI：

- 开始菜单
- 关卡选择或继续游戏按钮
- 游戏 HUD
- 商店按钮
- 金币数量
- 水晶核心购买进度
- 入侵警报提示
- 通关结算
- 失败结算

## 8. 胜负条件

### 胜利

购买 3 个水晶核心后通关。

### 失败

鱼缸内所有鱼死亡，且玩家金币不足以购买新鱼。

## 9. MVP 不做内容

为了控制范围，第一版不做：

- Boss 战
- 超过 3 种鱼
- 超过 3 种助手
- 复杂装饰系统
- 排行榜
- 广告与内购
- 云存档
- 小程序发布
- 多语言
- 复杂剧情

## 10. 美术方向

建议使用原创的轻量卡通风格，避免模仿《疯狂水族馆》的角色外形、UI 风格和配色。

可选方向：

- 扁平卡通水族箱
- 像素风鱼缸
- 柔和泡泡风
- 海底实验室风

MVP 阶段可先使用临时几何图形或自制占位图，优先验证手感。

## 11. 玩法调研后的后续优化建议

基于水族箱经营、点击收集和即时防守类玩法的调研，后续优先围绕“目标清晰、压力节奏、点击减负”继续迭代：

1. **多通道水晶提醒**：当金币足够购买水晶时，不只按钮变亮，还应有短时闪光、标签和轻微脉冲，提醒玩家这是通关目标。当前 HUD 会持续显示动态下一步提示，根据鱼群、敌人、金币和水晶价格提醒玩家当前优先行动。
2. **水晶进度可视化**：在鱼缸内或 HUD 中显示 3 个水晶槽位，让玩家比纯文字 `水晶：1/3` 更直观看到通关进度。
3. **入侵前预警窗口**：敌人出现前 1-2 秒给边缘红光或提示，让玩家从经营模式切换到防守模式，而不是突然被打断。
4. **防守期降低饥饿压力**：敌人在场时可让鱼饥饿消耗减半，避免“鱼饿死 + 敌人吃鱼”双重压力同时惩罚玩家。
5. **点击减负**：中后期加入金币磁吸、长按收集或连击收集，保留主动操作感，但降低精确点击疲劳。当前已支持按住鼠标或触摸拖过金币进行连续收集；玩家主动连续收金币会累积短时连击奖励，最高获得 40% 金币加成；点偏金币但附近有金币时会触发短距离磁吸并按主动收集结算。
6. **助手滚动解锁**：每引入一个新压力源，就自动化一个旧操作。当前第 1 关通关解锁清洁螺自动收金币，第 2 关通关解锁泡泡海马自动给饥饿鱼投喂，第 3 关通关解锁电光水母自动电击入侵敌人。
7. **经营-防守-奖励节奏**：关卡节奏应形成“经营期 → 入侵预警 → 防守期 → 战利品收集期”的循环，而不是持续不断施压。当前实现为击退最后一个敌人后进入短暂安全奖励窗口，暂停下一波倒计时，并让成熟鱼金币产出临时提高。

这些属于通用设计模式借鉴，不直接复刻任何现有游戏的角色、美术、音效、数值或 UI 表达。
