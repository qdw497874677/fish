# 《水族守卫》第一版开发任务拆解

## 1. 总体目标

第一版目标是使用 Godot 做出一个可在网页运行的 2D 水族箱经营 + 防守 Demo。

验收标准：

- 可以打开网页进入游戏
- 可以投喂鱼
- 鱼会吃食物、成长、产金币
- 玩家可以点击收金币
- 可以购买鱼和通关目标
- 敌人会定时入侵并攻击鱼
- 玩家可以点击攻击敌人
- 买满 3 个水晶核心后通关
- 鱼全灭且无法购买新鱼时失败
- 至少有 3 个关卡
- 至少解锁 1 个助手

## 2. 里程碑 0：项目初始化

目标：建立 Godot 项目并确认 Web 导出可用。

任务：

- [ ] 创建 Godot 4.x 项目
- [ ] 设置 2D 项目基础分辨率，建议 1280x720
- [ ] 设置拉伸模式，兼容不同浏览器窗口
- [ ] 创建基础目录结构
- [ ] 创建 MainMenu、AquariumLevel、ResultScreen 场景
- [ ] 安装 Web Export Templates
- [ ] 导出一个空 Web 页面并在本地浏览器运行

验收：

- 本地可以运行 Godot 项目
- 本地可以导出 Web 并打开空场景

## 9. 技术风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| Web 导出体积过大 | 页面加载慢 | 控制图片和音频体积 |
| 浏览器音频自动播放限制 | BGM 不播放 | 开始按钮点击后再播放 |
| 移动端性能不足 | 掉帧 | 控制鱼、金币、粒子数量 |
| 触摸误操作 | 体验差 | UI 区和玩法区输入优先级清晰 |
| 小程序迁移困难 | 后续平台风险 | MVP 不承诺小程序，单独评估 |

## 10. 下一步建议

下一步应先执行“里程碑 0：项目初始化”。完成后不要马上做完整 UI，先实现 AquariumLevel 中的基础水族箱原型。

推荐开发顺序：

```text
Godot 项目初始化
→ Web 空项目导出验证
→ 鱼随机游动
→ 投喂和吃食物
→ 掉金币和收金币
→ 商店购买
→ 敌人入侵
→ 通关失败
→ 关卡/助手
→ Web Demo 发布
```
