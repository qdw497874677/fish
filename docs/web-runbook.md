# Web 运行与打包说明

当前 Web 包使用手动包装方式：

- `index.html` 加载 `godot.js`
- `GODOT_CONFIG.executable` 设置为 `"godot"`
- Godot Web runtime 会按该名称加载 `godot.pck` 和 `godot.wasm`

因此资源包必须命名为：

```text
build/web/godot.pck
```

不要导出为 `game.pck`，否则网页会报：

```text
Failed loading file 'godot.pck'
```

## 本地运行

在项目根目录执行：

```bash
python3 -m http.server 8787 --directory build/web
```

然后打开：

```text
http://127.0.0.1:8787/index.html
```

## 重新打包 PCK

修改 Godot 脚本或场景后，重新导出：

```bash
godot --headless --path . --export-pack Web "build/web/godot.pck"
```

## Web 目录应包含

```text
build/web/
├── index.html
├── godot.pck
├── godot.js
├── godot.wasm
├── godot.audio.worklet.js
├── godot.audio.position.worklet.js
├── godot.service.worker.js
└── godot.offline.html
```

## 中文字体

Web 环境不会自动提供 Godot 默认中文字体。项目已内置：

```text
assets/fonts/PingFangSC-Regular.ttf
```

脚本会在运行时通过 Godot 的 `load("res://assets/fonts/PingFangSC-Regular.ttf")` 加载导入后的字体资源，并应用到 HUD、按钮和画布绘制文字。修改脚本或字体后，需要重新执行：

```bash
godot --headless --path . --export-pack Web "build/web/godot.pck"
```

如果浏览器仍显示旧乱码，先强制刷新：

```text
Cmd + Shift + R
```

## 本地进度存档

游戏会保存长期进度，不保存当前局内金币、鱼群和敌人状态。当前支持 3 个存档槽，每个槽保存：

- 存档名称
- 最高已解锁关卡
- 已通关关卡
- 清洁螺是否已解锁
- 累计游玩时间

Godot 使用：

```text
user://aquarium_guard_save.json
```

Web 版会由浏览器持久化该 `user://` 数据。清除站点数据或浏览器缓存时，所有槽位进度可能被重置。旧版单存档会自动迁移为第 1 个槽位，名称为 `旧存档`。

## 清洁螺助手

第 1 关通关后会解锁清洁螺。清洁螺会在鱼缸底部附近巡逻，发现金币后主动移动过去，靠近金币时自动收集。没有金币时，它会回到底部起点等待。

## 开始菜单与关卡选择

网页启动后会先进入开始菜单，而不是直接进入关卡。菜单提供：

- `继续第 N 关`：从当前选中存档的最高已解锁关卡开始
- 第 1-3 关选择按钮：根据当前选中存档决定是否解锁，已通关关卡会显示 `✓`
- `存档管理`：显示 3 个存档槽；底部统一输入名称并点击 `新建存档`，会自动占用第一个空槽；已有槽可以进入或删除

游戏 HUD 右侧的 `菜单` 按钮可以随时回到开始菜单。返回菜单会清空当前局内鱼、金币、敌人和食物，但不会清除长期进度存档。

## 失败与计时

关卡没有时间限制，但 HUD 会显示当前存档的累计游玩时间，并写入对应存档槽。

进入关卡时会显示本关目标提示；通关或失败时会显示本局统计，包括用时、击败敌人、损失鱼、金币收入、购买鱼和最高鱼数。

失败规则：

- 如果鱼群全灭且金币不足以购买最便宜的鱼，会立即失败。
- 如果鱼群全灭但金币足够购买至少一条鱼，会进入 12 秒补救倒计时。
- 倒计时内购买新鱼会继续游戏；倒计时结束仍没有鱼则失败。

## 鱼类型

关卡 HUD 顶部中间常驻 `快捷购买` 栏，不需要打开商店，也不会遮挡鱼缸。购买栏直接显示鱼、食物和水晶的短按钮，点击对应商品即可购买；没钱或暂停时按钮会自动置灰。

快捷购买栏支持直购三种鱼：

- 蓝泡鱼：80 金币，便宜稳定，成长和产钱均衡。
- 金鳞鱼：150 金币，成长略慢，但成熟后单次产金币更多。
- 护卫鱼：120 金币，成长更快、体型更大，产金币较少；成熟后会自动攻击附近敌人，适合提高防守容错。

## 敌人与波次

HUD 会显示 `下一波：N秒`，用于提示下一次敌人入侵节奏。

当前敌人类型：

- 小章怪：基础敌人，会追鱼并吃鱼。
- 甲壳怪：第 3 关开始更常见，血量更高、速度更慢。
- 偷金币怪：第 2 关开始出现，优先追金币并偷走金币；没有金币时才回到普通追鱼压力。

## 手机浏览器兼容

当前 `build/web/index.html` 针对手机浏览器做了基础处理：

- `viewport-fit=cover` 适配刘海屏/安全区
- 使用 `100dvh` 避免地址栏收起/展开导致画布高度异常
- 禁止页面滚动、双指/双击缩放和长按菜单
- Godot 顶部 HUD 高度提升到 96px，按钮高度提升到 56px，方便触摸

建议横屏体验。如果竖屏打开，浏览器会等比缩放画布，但操作空间会比较紧。

## 暂停

游戏 HUD 右侧有 `暂停` 按钮。暂停后：

- 鱼、金币、敌人、食物和命中特效都会停止更新
- 水体点击不会投喂、收金币或攻击敌人
- 顶部按钮会切换为 `继续`
- 可以点击 `菜单` 放弃当前局并返回关卡选择

## 当前难度基准

当前 Web Demo 使用中等新手难度：

- 第 1 关初始 230 金币、3 条鱼
- 最低鱼价 80 金币，顶部快捷购买栏可直接购买蓝泡鱼、金鳞鱼、护卫鱼
- 第 1 关水晶总成本约 1350 金币
- 鱼仍比早期版本更耐饿，但需要持续投喂
- 不同鱼成熟后的金币产出不同：蓝泡鱼均衡，金鳞鱼收益高，护卫鱼收益低但成长快
- 第 1 关敌人约 20 秒后首次出现，小章怪需要点击 5 次击败

如果测试仍觉得太简单，可以继续提高 `core_base_cost` / `core_step_cost`，或缩短 `enemy_timer`；如果觉得太难，则反向调整。
