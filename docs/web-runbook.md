# Web 运行与打包说明

当前 Web 包使用手动包装方式：

- `index.html` 加载 `godot.js`
- `GODOT_CONFIG.executable` 设置为 `"godot"`
- Godot Web runtime 会按该名称加载 `godot.pck` 和 `godot.wasm`

因此资源包必须命名为：

```text
build/web/godot.pck
```

不要导出为 `game.pck`，否则网页会报：

```text
Failed loading file 'godot.pck'
```

## 本地运行

在项目根目录执行：

```bash
python3 -m http.server 8787 --directory build/web
```

然后打开：

```text
http://127.0.0.1:8787/index.html
```

## 重新打包 PCK

修改 Godot 脚本或场景后，重新导出：

```bash
godot --headless --path . --export-pack Web "build/web/godot.pck"
```

## 当前脚本分层

项目仍以 `scenes/aquarium_level.tscn` 作为主场景，但核心逻辑已开始从单脚本渐进拆分：

后续开发必须遵守 `docs/development-guidelines.md` 中的分层、验证和 Web 导出规范。

- `scripts/aquarium_level.gd`：关卡总控、主循环、输入、HUD/menu 连接、实体更新与绘制调度。
- `scripts/data/game_data.gd`：关卡配置和鱼类型配置。
- `scripts/systems/audio_system.gd`：运行时合成音效、背景氛围音、音效播放池和开关状态。
- `scripts/systems/save_system.gd`：存档槽数据模型、归一化、槽位查找，以及 `user://aquarium_guard_save.json` 读写封装。
- `scripts/gameplay/aquarium_queries.gd`：玩法层几何与最近目标查询工具，负责最近鱼/食物/敌人/金币索引、鱼群分离向量、随机水域位置和边界夹取。
- `scripts/gameplay/fish_logic.gd`：鱼实体的纯逻辑工具，负责鱼字典创建、觅食局部更新、护卫鱼目标资格判断和护卫移动速度计算。
- `scripts/gameplay/enemy_logic.gd`：敌人实体的纯逻辑工具，负责敌人字典创建、追鱼/漂移/偷金币移动、攻击与偷金币距离判断、攻击冷却重置。
- `scripts/gameplay/resource_logic.gd`：资源实体的纯逻辑工具，负责食物/金币字典创建、下沉与生命周期判断、玩家金币磁吸、成熟鱼金币出生点、清洁螺目标选择和位移计算。
- `scripts/gameplay/combat_logic.gd`：战斗规则工具，负责点击命中敌人、敌人扣血/击败判断，以及敌人金币奖励计算。
- `scripts/gameplay/wave_logic.gd`：波次与防守状态工具，负责提示计时、刷怪计时、下一波随机间隔、入侵预警和防守压力状态判断。
- `scripts/gameplay/effect_logic.gd`：视觉反馈数据工具，负责命中特效/护卫射线特效字典创建、生命周期更新和震屏参数。
- `scripts/gameplay/economy_logic.gd`：经济规则工具，负责投喂/升级/水晶成本、购买条件和最低鱼价计算。
- `scripts/gameplay/combo_logic.gd`：连击规则工具，负责玩家主动收集金币时的连击窗口、连击倍率和额外金币计算。
- `scripts/gameplay/progression_logic.gd`：进度规则工具，负责无鱼失败倒计时、通关解锁下一关、助手解锁、水晶通关判断，以及局内动态下一步提示推导。
- `scripts/ui/aquarium_ui_factory.gd`：无状态 UI 控件工厂，统一创建 `Panel`、`Label`、`Button`、`LineEdit` 并应用中文字体和按钮焦点策略。
- `scripts/ui/aquarium_hud_presenter.gd`：HUD 纯展示模型计算，负责金币/状态栏文案、购买按钮文案与禁用状态、暂停/重开按钮文案。

每次继续拆分后，至少运行：

```bash
godot --headless --path . --quit-after 120
```

涉及 Web 包发布或脚本/场景最终验证时，再运行：

```bash
godot --headless --path . --export-pack Web "build/web/godot.pck"
```

## 中文字体

Web 环境不会自动提供 Godot 默认中文字体。项目已内置：

```text
assets/fonts/PingFangSC-Regular.ttf
```

脚本会在运行时通过 Godot 的 `load("res://assets/fonts/PingFangSC-Regular.ttf")` 加载导入后的字体资源，并应用到 HUD、按钮和画布绘制文字。修改脚本或字体后，需要重新执行：

```bash
godot --headless --path . --export-pack Web "build/web/godot.pck"
```

如果浏览器仍显示旧乱码，先强制刷新：

```text
Cmd + Shift + R
```

## 本地进度存档

游戏会保存长期进度，不保存当前局内金币、鱼群和敌人状态。当前支持 3 个存档槽，每个槽保存：

- 存档名称
- 最高已解锁关卡
- 已通关关卡
- 清洁螺是否已解锁
- 泡泡海马是否已解锁
- 电光水母是否已解锁
- 累计游玩时间

Godot 使用：

```text
user://aquarium_guard_save.json
```

Web 版会由浏览器持久化该 `user://` 数据。清除站点数据或浏览器缓存时，所有槽位进度可能被重置。旧版单存档会自动迁移为第 1 个槽位，名称为 `旧存档`。

## 清洁螺助手

第 1 关通关后会解锁清洁螺。清洁螺会在鱼缸底部附近巡逻，发现金币后主动移动过去，靠近金币时自动收集。没有金币时，它会回到底部起点等待。

## 泡泡海马助手

第 2 关通关后会解锁泡泡海马，并写入当前存档。后续进入任意已解锁关卡时，泡泡海马会平时待在鱼缸右上方；当鱼群里有饥饿鱼且场上食物不多时，它会先游到饥饿鱼上方，再吐出一次免费的低一级泡泡鱼食，帮助玩家降低中后期反复投喂压力。它不会扣除金币，也不会在食物已经较多时继续堆叠。

## 电光水母助手

第 3 关通关后会解锁电光水母，并写入当前存档。后续进入任意已解锁关卡时，电光水母会漂浮在鱼缸上方；只要场上存在敌人，它会按固定间隔自动选择附近敌人释放电弧，造成 1 点伤害。若这次电击击败敌人，会正常掉落金币并触发安全奖励窗口。

## 开始菜单与关卡选择

网页启动后会先进入开始菜单，而不是直接进入关卡。菜单提供：

- `继续第 N 关`：从当前选中存档的最高已解锁关卡开始
- 第 1-3 关选择按钮：根据当前选中存档决定是否解锁，已通关关卡会显示 `✓`
- `存档管理`：显示 3 个存档槽；底部统一输入名称并点击 `新建存档`，会自动占用第一个空槽；已有槽可以进入或删除

游戏 HUD 右侧的 `菜单` 按钮可以随时回到开始菜单。返回菜单会清空当前局内鱼、金币、敌人和食物，但不会清除长期进度存档。

## 失败与计时

关卡没有时间限制，但 HUD 会显示当前存档的累计游玩时间，并写入对应存档槽。

进入关卡时会显示本关目标提示；局内 HUD 下方会持续显示动态下一步提示，例如优先补鱼、投喂、收金币、防守或购买水晶；通关或失败时会显示本局统计，包括用时、击败敌人、损失鱼、金币收入、购买鱼和最高鱼数。

鱼缸底部会显示 3 格 `水晶核心` 进度槽：已购买的水晶会发光填充，下一颗水晶会显示价格；当金币接近或足够购买下一颗水晶时，对应槽位会发光提醒，和顶部 `水晶` 快捷按钮形成双重提示。

金币可以单次点击收集，也可以按住鼠标左键或触摸并在水体里拖过金币来连续收集。按下瞬间仍保留原有优先级：先攻击敌人，再收金币，最后才投喂；拖动过程中只会收金币，不会连续投喂。点击点没有直接命中金币但附近有金币时，会短暂把附近金币吸向点击点，到达后按玩家主动收集结算。玩家主动连续收金币会在短时间内累积连击倍率，HUD 会显示当前 `收金币连击` 和加成；清洁螺的自动收金币不参与该主动连击奖励。

失败规则：

- 如果鱼群全灭且金币不足以购买最便宜的鱼，会立即失败。
- 如果鱼群全灭但金币足够购买至少一条鱼，会进入 12 秒补救倒计时。
- 倒计时内购买新鱼会继续游戏；倒计时结束仍没有鱼则失败。

## 鱼类型

关卡 HUD 顶部中间常驻 `快捷购买` 栏，不需要打开商店，也不会遮挡鱼缸。购买栏直接显示鱼、食物和水晶的短按钮，点击对应商品即可购买；没钱或暂停时按钮会自动置灰。

快捷购买栏支持直购三种鱼：

- 蓝泡鱼：80 金币，便宜稳定，成长和产钱均衡。
- 金鳞鱼：150 金币，成长略慢，但成熟后单次产金币更多。
- 护卫鱼：120 金币，成长更快、体型更大，产金币较少；成熟后会优先靠近敌人并自动攻击附近敌人，适合提高防守容错。没有敌人时才恢复普通吃食和游动。

## 敌人与波次

HUD 会显示 `下一波：N秒`，用于提示下一次敌人入侵节奏。

敌人出现前最后 2 秒会进入预警状态：HUD 文案切换为 `入侵预警`，鱼缸边缘出现红色脉冲，并在画面上方显示倒计时。敌人真正出现后再显示具体入侵提示。

敌人在场时会进入防守期，鱼的饥饿消耗降为平时的一半；这样玩家可以先处理敌人，击退后再回到投喂和经营节奏。

击退当前场上最后一个敌人后，会进入短暂 `安全奖励` 窗口。窗口期间下一波刷怪倒计时暂停，鱼的饥饿仍按防守期减缓，成熟鱼产金币临时提高 20%，HUD 会显示剩余秒数。

当前敌人类型：

- 小章怪：基础敌人，会追鱼并吃鱼。
- 甲壳怪：第 3 关开始更常见，血量更高、速度更慢。
- 偷金币怪：第 2 关开始出现，优先追金币并偷走金币；没有金币时才回到普通追鱼压力。

## 手机浏览器兼容

当前 `build/web/index.html` 针对手机浏览器做了基础处理：

- `viewport-fit=cover` 适配刘海屏/安全区
- 使用 `100dvh` 避免地址栏收起/展开导致画布高度异常
- 禁止页面滚动、双指/双击缩放和长按菜单
- Godot 顶部 HUD 高度提升到 96px，按钮高度提升到 56px，方便触摸
- 首次触摸局内水体时会短暂显示非阻断提示，建议横屏并说明点敌人、点金币、拖拽连收

建议横屏体验。如果竖屏打开，浏览器会等比缩放画布，但操作空间会比较紧。

## 暂停

游戏 HUD 右侧有 `暂停` 按钮。暂停后：

- 鱼、金币、敌人、食物和命中特效都会停止更新
- 水体点击不会投喂、收金币或攻击敌人
- 顶部按钮会切换为 `继续`
- 可以点击 `菜单` 放弃当前局并返回关卡选择

## 音频

游戏顶部 HUD 右侧提供 `音效开` / `音效关` 按钮，会统一控制背景氛围音和所有操作音效。当前音频不依赖外部文件，而是在 Godot 运行时生成轻量合成音，避免增加 Web 包资源加载风险。

当前已有音效反馈：

- 投喂鱼食
- 收集金币，包括清洁螺自动收集
- 购买鱼、升级食物、购买水晶
- 敌人入侵预警
- 点击攻击、电光水母自动电击、击败敌人、鱼被敌人吃掉
- 通关和失败结算

浏览器可能要求玩家先点击页面后才允许播放音频；如果首次进入菜单没有背景音，点击任意游戏按钮或切换一次 `音效开` 即可激活。

## 当前难度基准

当前 Web Demo 使用中等新手难度：

- 第 1 关初始 230 金币、3 条鱼
- 最低鱼价 80 金币，顶部快捷购买栏可直接购买蓝泡鱼、金鳞鱼、护卫鱼
- 第 1 关水晶总成本约 1350 金币
- 鱼仍比早期版本更耐饿，但需要持续投喂
- 第 2 关通关后，泡泡海马会自动给饥饿鱼补充少量食物，降低后续关卡投喂压力
- 第 3 关通关后，电光水母会周期性自动电击敌人，降低重玩关卡时的点击攻击压力
- 支持按住/触摸拖过金币连续收集，点偏时可触发附近金币磁吸，并通过主动收金币连击获得最高 40% 金币加成，降低中后期精确点击疲劳
- 手机端首次触摸局内水体时会显示 6 秒触控提示，不阻塞操作，返回菜单或重开关卡后可再次触发
- 不同鱼成熟后的金币产出不同：蓝泡鱼均衡，金鳞鱼收益高，护卫鱼收益低但成长快
- 第 1 关敌人约 20 秒后首次出现，小章怪需要点击 5 次击败

如果测试仍觉得太简单，可以继续提高 `core_base_cost` / `core_step_cost`，或缩短 `enemy_timer`；如果觉得太难，则反向调整。
